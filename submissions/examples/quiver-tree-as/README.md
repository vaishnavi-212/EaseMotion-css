# Quiver Tree

A pure CSS/HTML *Aloidendron dichotomum* on the Namaqualand veld, generated by the branching rule it is named for.

## What it shows

The quiver tree is a giant aloe, and its name is a description of its geometry: **dichotomum**, forking in two.

Every branch point splits into **exactly two**, and both children are roughly equal. There is no main leader carrying on past the fork with side branches hanging off it, which is how most trees work. So the tree doubles: 1 trunk, then 2, 4, 8, 16, 32. That single rule is what produces the distinctive evenly-domed crown, with a rosette of thick leaves sitting at every one of the tips and nowhere else.

The other things about it are all consequences of living where nothing should:

- **The trunk is a water tank**, spongy and fibrous rather than woody.
- **The bark peels into thin plates with genuinely sharp edges.** It is a real hazard to handle.
- **The name is from the San**, who hollowed the branches to make quivers. The soft fibrous core that makes it a bad tree makes it a good tube.
- **Sociable weavers build in it** because it is often the only structure for kilometres. The nests get big enough to break the limb they are on.

## How it is built

- **The tree is generated, not drawn.** A recursive function forks at every tip, five deep, with each child at 72% of its parent's length. The output is 63 limbs and 32 crowns.
- **The doubling is verified.** The browser check counts limbs per depth straight off the DOM and gets **1, 2, 4, 8, 16, 32**, confirming each level is exactly twice the one above it. Dichotomous branching is the species' defining trait, so a tree that merely looked bushy would be the wrong plant.
- **Every fork actually joins.** The check recomputes each limb's tip from its own `--x`, `--y`, `--len` and `--a`, then measures the distance to its children's start points: **zero orphan limbs**, worst joint gap **0.009px**. Nothing floats.
- **Each limb keeps its own angle** through the sway, because the shared keyframe rebuilds it with `rotate(calc(var(--a) * -1 + 1.2deg))` rather than overwriting it.
- **Rosettes only at the tips**, which is where an aloe puts leaves, drawn with a `repeating-conic-gradient` and masked to a disc.
- **The bark is drawn as plates**: a `repeating-linear-gradient` scale layer on every limb.

No images, no JavaScript, no external assets.

## Accessibility

`prefers-reduced-motion: reduce` disables every keyframe. The tree is a structure, so nothing is lost.
