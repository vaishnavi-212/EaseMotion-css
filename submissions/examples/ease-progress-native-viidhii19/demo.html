Zero-JS Scroll-Driven Reading Progress Bar

This page demonstrates a high-performance reading progress bar that requires absolute zero JavaScript.

Scroll down to see the progress bar at the top of the page smoothly track your scroll position. It uses the modern CSS @scroll-timeline API to run entirely on the browser's compositor thread.

Section 1: The Problem with JS Scroll Listeners

Article reading progress bars almost exclusively use JavaScript onScroll event listeners. These listeners poll the DOM continuously to calculate the scroll percentage and update the progress bar width accordingly.

This approach has a few major drawbacks. First, it runs on the main thread, meaning it competes with other JavaScript tasks for resources. Second, it can cause layout thrashing if not implemented carefully. Third, it degrades page performance, especially on lower-end devices or heavily loaded pages.

Section 2: The Native CSS Solution

With the introduction of the modern CSS animation-timeline: scroll() API, we can now link animations directly to the scroll position of a scroll container (like the root block).

This means we can create a reading progress bar using only CSS, completely eliminating the need for JavaScript scroll listeners. The animation runs strictly on the browser's compositor thread, resulting in a buttery smooth, hardware-accelerated experience that doesn't block the main thread.
