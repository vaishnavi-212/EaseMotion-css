Hinge Animation Stacking Context Issue

This demo illustrates how the `hinge` animation can inadvertently render behind surrounding content due to stacking context rules.

Card A (z-index: 1, transformed)

This card creates a new stacking context. The animated element is inside.

Card B (z-index: 2)

Because Card B has a higher z-index than Card A, the hinging element from Card A will fall behind this card, even though the hinging element has `z-index: 9999`.
