Parser Input Bounds Validation Fix

Bug A — CRLF Injection

em="fade-in\r\nbounce 999ms" causes the parser to pick up bounce as the animation (from the second line), completely ignoring fade-in.

Bug B — Unbounded Duration (DoS)

em="fade-in 999999999ms" generates a 277-hour animation. No upper bound check exists, so animationend events never fire.

Bug C — Unbounded Iteration Count

em="bounce repeat-9999999999" generates animation-iteration-count: 9999999999, which exceeds CSS float precision. Browsers silently ignore it and fall back to 1 iteration — breaking the developer's intent with no error.

See README.md and parser-fix.js for details.
