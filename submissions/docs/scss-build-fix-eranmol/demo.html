SCSS Build Output Fix

Issue #40201 — build:scss emits only a source map comment

Problem

npm run build:scss compiles scss/_index.scss, which only contains @forward statements for variables and mixins. This produces an empty CSS file with just a source map comment:

/*# sourceMappingURL=easemotion.scss.css.map */

Root Cause

FileContentsIssue
scss/_index.scss @forward 'variables';
@forward 'mixins';		 Only forwards partials — no CSS output
scss/_variables.scss CSS custom properties Variables only, no selectors
scss/_mixins.scss SCSS mixins Mixins only, no selectors

Fix

Create a new entry file scss/easemotion.scss that @uses the index and adds actual CSS rules, then update the build script to target it.

1. New file: scss/easemotion.scss

// EaseMotion CSS — SCSS Build Entry
@use 'index';

// Import core CSS files via sass built-in
@use '../core/variables';
@use '../core/base';
@use '../core/animations';
@use '../core/utilities';

// Import component CSS files
@use '../components/buttons';
@use '../components/cards';
@use '../components/chips';
@use '../components/forms';
@use '../components/modals';
@use '../components/navigation';
@use '../components/pagination';
@use '../components/progress';
@use '../components/tables';
@use '../components/tooltips';

2. Update package.json (line 34)

- "build:scss": "sass scss/_index.scss dist/easemotion.scss.css"
+ "build:scss": "sass scss/easemotion.scss dist/easemotion.scss.css"

Alternative (Minimal Fix)

If the goal is only to export SCSS variables/mixins for library consumers (not generate CSS), change the output file extension to reflect that:

"build:scss": "sass scss/_index.scss dist/easemotion.scss.css --no-source-map"

And document that this file is for SCSS import only, not for use as a CSS stylesheet.
