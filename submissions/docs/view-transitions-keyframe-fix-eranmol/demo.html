View Transitions Keyframe Prefix Fix

Issue #40209 — Keyframes should use ease-kf-* convention

Problem

components/view-transitions.css defines keyframes with generic view-* names that risk collision with consumer projects:

@keyframes view-fade-out   /* ❌ generic */
@keyframes view-fade-in    /* ❌ generic */
@keyframes view-scale-out  /* ❌ generic */
@keyframes view-scale-in   /* ❌ generic */

Expected Convention

All framework keyframes should use the ease-kf-* prefix, matching the pattern used in core/animations.css:

@keyframes ease-kf-view-fade-out   /* ✅ prefixed */
@keyframes ease-kf-view-fade-in    /* ✅ prefixed */
@keyframes ease-kf-view-scale-out  /* ✅ prefixed */
@keyframes ease-kf-view-scale-in   /* ✅ prefixed */

Fix — Updated File

Replace the contents of components/view-transitions.css with:

@layer easemotion-components {
  @keyframes ease-kf-view-fade-out {
    to { opacity: 0; }
  }

  @keyframes ease-kf-view-fade-in {
    from { opacity: 0; }
  }

  @keyframes ease-kf-view-scale-out {
    from { transform: scale(1); border-radius: 0; }
    to   { transform: scale(0.95); border-radius: 20px; }
  }

  @keyframes ease-kf-view-scale-in {
    from { transform: scale(0.95); border-radius: 20px; }
    to   { transform: scale(1); border-radius: 0; }
  }
}
