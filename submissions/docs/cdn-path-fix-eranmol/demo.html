CDN Path Consistency Fix

Issue #47282 — Unify CDN paths in docs/index.html

Problem

docs/index.html (lines 324–338) shows two different CDN URL patterns for the same file:

LineCurrent PathSource
325 cdn.jsdelivr.net/gh/SAPTARSHI-coder/EaseMotion-css@main/easemotion.min.css GitHub-based
332 cdn.jsdelivr.net/npm/easemotion-css/easemotion.min.css npm-based

These may resolve to different versions. Users following the "recommended" path get the GitHub version; those copying the "alternative" get the npm version.

Fix

Standardize on the npm-based jsDelivr path as the single recommended CDN URL. It always resolves to the latest published npm package version and is the most reliable for production use.

Replace the "Option 1" code block (lines 324–326)

- <!-- CDN via jsDelivr (GitHub) — recommended, works everywhere -->
- <link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/SAPTARSHI-coder/EaseMotion-css@main/easemotion.min.css" />
+ <!-- CDN via jsDelivr (npm) — recommended, works everywhere -->
+ <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css@latest/easemotion.min.css" />

Update the alternatives section (lines 331–338)

 <!-- Alternative CDN providers -->

+<!-- GitHub Raw ( pinned to main ) -->
+<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/SAPTARSHI-coder/EaseMotion-css@main/easemotion.min.css" />
+
 <!-- unpkg -->
 <link rel="stylesheet" href="https://unpkg.com/easemotion-css@latest/easemotion.min.css" />

-<!-- GitHub Raw CDN -->
-<link rel="stylesheet" href="https://raw.githubusercontent.com/SAPTARSHI-coder/EaseMotion-css/main/easemotion.min.css" />

Result

After the fix, the docs will show a single recommended path (npm) with two clear alternatives (GitHub pinned, unpkg). No conflicting "recommended" labels.
